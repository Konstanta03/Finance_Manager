# Finance Manager 2.0 — коротко для захисту

## Реалізовано

- Реєстрація та вхід користувачів через ASP.NET Core Identity.
- Ролі користувачів: `User` та `Admin`.
- Стартовий адміністратор створюється автоматично.
- Додавання, редагування, видалення та перегляд транзакцій.
- Фільтрація транзакцій за нотаткою, сумою, датою та категорією.
- Сортування транзакцій за датою, сумою та категорією.
- Порівняльна статистика витрат за різні періоди.
- Планувальник регулярних платежів.
- Налаштування користувача: валюта, мова, тема.
- Перемикання світлої та темної теми.
- Health-check для деплою: `/health`.
- Unit-тести винесені в окремий test-проєкт.

## Архітектура

Проєкт побудований за схемою MVC:

- `Controllers` — приймають запити користувача.
- `Models` — описують сутності бази даних.
- `Views` — відповідають за HTML-інтерфейс.
- `Services` — містять бізнес-логіку, винесену з контролерів.
- `ViewModels` — використовуються для форм входу, реєстрації та фільтрації.
- `Tests/FinanceManager.Tests` — окремий проєкт для unit-тестів.

## Identity

Реєстрація реалізована через `UserManager<ApplicationUser>`, а вхід — через `SignInManager<ApplicationUser>`. Після реєстрації користувач автоматично отримує роль `User` і входить у систему.

## Тестування

Тести не використовують реальну PostgreSQL базу. Для перевірки логіки застосовуються mock-и та InMemory-підхід, тому тести можна запускати незалежно від локальної бази даних.

Команда:

```bash
dotnet test Tests/FinanceManager.Tests/FinanceManager.Tests.csproj
```
